# Improvement Index

This directory contains one markdown artifact per tracked improvement.

Use these files to record:

- the current gap
- why the gap matters
- the intended outcome
- constraints and dependencies
- progress notes
- the acceptance boundary for marking the improvement complete

Backlog and in-progress items live in `candidates/`; completed items live in `done/`. Non-candidate exploration is recorded in [undiscovered-improvements.md](../undiscovered-improvements.md).
